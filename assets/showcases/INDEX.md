## Usage

### Referencing Samples in Phase 3
After recommending a design direction, show the ready-made screenshot for the matching scenario:
```
"This is how the Pentagram direction looks as a social-media cover → [show cover/cover-pentagram.png]"
"Here is the Takram direction applied to a PPT data slide → [show ppt/ppt-takram.png]"
```

### Scenario-Matching Priority
1. Exact match for the user's scenario → show that scenario directly.
2. No exact match, but a related type exists → show the closest scenario (for example, "product website" → SaaS landing page).
3. No meaningful match → skip the ready-made samples and generate a bespoke example in Phase 3.5.

### Side-by-Side Comparison
Show all three styles for one scenario side by side so the user can compare them directly:
- "Here is the same social-media cover rendered in three distinct styles."
- Display order: Pentagram (rational restraint) → Build (luxury minimalism) → Takram (gentle warmth).
